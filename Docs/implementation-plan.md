# 持久上下文系统 - 实现方案

## 1. 项目概述

### 1.1 目标
创建一个可复用的持久上下文系统模板，使 AI 能够自主维护 CLAUDE.md 文件，实现跨会话的项目知识持久化。

### 1.2 核心理念
- **SKILL 为核心**：SKILL 自包含，即使没有 Hook 也能完整工作
- **Hook 为辅助**：Prompt 类型 Hook 用于提醒 AI 执行 SKILL
- **增量更新**：文档更新采用增量策略，保持一致性

---

## 2. 目录结构设计

```
项目根目录/
├── CLAUDE.md                  # 主记忆文件（Claude Code 自动加载）
│
└── Docs/                      # 详细文档目录（树状结构）
    ├── index/                 # 索引目录
    │   ├── _tree.json         # 结构化索引（程序处理）
    │   └── index.md           # Markdown 索引（便于阅读）
    ├── api/                   # API 相关文档
    │   └── *.md
    ├── modules/               # 模块相关文档
    │   └── *.md
    └── decisions/             # 架构决策记录
        └── *.md
```

## 3. CLAUDE.md 内容规范

按优先级排列：

### 3.1 约束条件/禁止项（最高优先级）
- 禁止修改的位置
- 必须遵守的规则
- 安全相关限制

### 3.2 项目偏好
- 项目特定的编码风格
- 框架选择和配置
- 技术栈约定

### 3.3 用户偏好
- 个人开发偏好设置
- 输出格式偏好
- 沟通风格偏好

### 3.4 常用工具/指令
- 频繁使用的命令
- 常用工具配置
- 快捷操作指南

### 3.5 项目结构概览
- 整体架构说明
- 目录结构说明
- 模块依赖关系

### 3.6 技术文档索引
- 指向 Docs/ 下详细文档的快速链接
- 按类别组织的文档导航

### 3.7 架构决策记录摘要
- 重要决策的简要说明
- 决策原因和影响

### 3.8 当前任务状态
- 记录当前工作进展
- 跨会话保持工作连贯性
- 待办、进行中、已完成的任务追踪
- 上次更新的具体内容

## 4. 索引映射关系

| 映射类型 | 说明 |
|----------|------|
| 函数名 → 文档路径 | 根据函数名查找对应技术文档 |
| 模块名 → 文档路径 | 根据模块名查找对应技术文档 |
| API端点 → 文档路径 | 根据 API 端点查找对应文档 |
| 文档摘要 → 文档路径 | 根据摘要内容查找对应文档 |

> 注：关键词 → 文档路径映射属于向量库功能，暂不实现

## 6. 文档写作规范

### 6.1 格式要求
- 使用固定模板（在 SKILL 中规定）
- Markdown 格式
- 清晰的章节结构

### 6.2 更新策略
- **增量更新**：只更新变更的部分，保留其他内容
- 保持文档一致性

### 6.3 索引存储
- **JSON 文件**：`_tree.json` 便于程序处理
- **Markdown 文件**：`index.md` 便于人工阅读

## 7. 组件设计

### 7.1 SKILL（核心）

**位置**：`~/.claude/skills/persistent-context/SKILL.md`

**功能**：
- 定义完整的文档更新流程
- 定义文档模板和写作规范
- 定义索引维护规则
- AI 可独立理解和执行

**SKILL 格式要求**：
```yaml
---
name: persistent-context
description: [SKILL 做什么] + [何时使用该 SKILL] + [触发该 SKILL 关键词]
origin: ECC
---
```

### 7.2 Rules（强制加载）

**位置**：`~/.claude/rules/memory-loading.md`

**功能**：
- 强制要求 AI 在会话开始时加载 CLAUDE.md
- 定义文档加载优先级

### 7.3 Hook（可选增强）

**类型**：`type: "prompt"` - 提醒文本

**触发时机**：

| Hook 类型 | 触发时机 | 提醒内容 |
|-----------|----------|----------|
| Stop Hook | 会话结束时 | "请检查是否需要更新项目文档，执行 /update-memory" |

**无 Hook 备选**：
- 用户手动使用 `/update-memory` 命令
- AI 根据 SKILL 自主判断

### 7.4 文档模板

**位置**：`templates/` 目录

| 模板文件 | 用途 |
|----------|------|
| `claude-md.template.md` | CLAUDE.md 主记忆文件模板 |
| `api-doc.template.md` | API 文档模板 |
| `module-doc.template.md` | 模块文档模板 |

---

## 8. 更新触发机制

### 8.1 自动触发（Stop Hook）
- 会话结束时，Hook 提醒 AI 检查是否需要更新
- AI 分析本次会话的变更
- 增量更新相关文档

### 8.2 手动触发
- `/update-memory`：手动触发文档更新
- `/record-refactor`：记录大型重构变更（用户判断）

### 8.3 无 Hook 场景
- AI 根据 SKILL 自主判断是否需要更新
- 用户手动触发命令

---

## 9. 大型重构记录

- **不自动检测**：由用户手动判断是否为大型重构
- **手动触发**：用户使用 `/record-refactor` 命令
- **记录内容**：
  - 重构范围
  - 影响的模块
  - 主要变更点
