# 持久上下文加载规则

> 此规则确保 AI 在会话开始时自动加载项目记忆，实现跨会话知识持久化。

---

## 会话开始行为

每个新会话开始时，AI **必须**执行以下操作：

### 1. 加载主记忆文件

```
必须读取: CLAUDE.md (项目根目录)
```

此文件包含：
- 项目约束和禁止项
- 项目和用户偏好
- 常用工具和指令
- 项目结构概览
- 技术文档索引
- 架构决策记录 (ADR)
- 当前任务状态

### 2. 加载文档索引（如存在）

```
如果存在则读取: Docs/index/index.md
```

此文件提供：
- 文档分类列表
- 各文档路径和描述
- 快速导航信息

---

## 文档加载优先级

当需要获取项目上下文时，按以下优先级加载：

| 优先级 | 文件 | 说明 |
|--------|------|------|
| 1 (最高) | `CLAUDE.md` | 主记忆文件，包含核心信息 |
| 2 | `Docs/index/index.md` | 文档索引，用于导航 |
| 3 | 具体 ADR 文档 | 架构决策详情 |
| 4 | 技术文档 | 按需加载的详细文档 |

---

## 上下文使用规范

### 信息查找顺序

1. **首先检查 CLAUDE.md** - 大部分信息应在此找到
2. **其次检查索引** - 确定详细文档位置
3. **最后加载具体文档** - 获取详细信息

### 避免重复加载

- 同一会话内不重复读取同一文件
- 使用已加载的上下文回答问题
- 仅在确认信息过时时重新加载

---

## 更新触发条件

以下情况 **必须** 考虑更新 CLAUDE.md：

- [ ] 新增重要功能或模块
- [ ] 技术栈发生变更
- [ ] 做出重要架构决策
- [ ] 项目结构发生变化
- [ ] 用户偏好发生变化
- [ ] 当前任务状态发生变化

更新时使用 `persistent-context` SKILL 定义的增量更新策略。

---

## 与 SKILL 的关系

此 **Rule** 定义 **必须** 加载哪些文件。

**SKILL** (`persistent-context`) 定义 **如何** 维护这些文件。

| 组件 | 职责 |
|------|------|
| Rule (本文件) | 强制加载行为，确保上下文可用 |
| SKILL | 维护工作流程，定义更新规则 |
| Hook | 提醒机制，辅助记忆持久化 |

---

## 验证检查

会话开始时，AI 应能回答：

1. ✅ 项目的约束条件是什么？
2. ✅ 用户偏好的文档语言是什么？
3. ✅ 当前正在进行什么任务？
4. ✅ 有哪些重要的架构决策？

如果无法回答以上问题，说明 CLAUDE.md 可能需要更新或不存在。

---
