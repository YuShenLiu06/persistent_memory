# 持久上下文管理 (Persistent Context)

## 适用场景

使用此 SKILL 当：

1. **需要更新项目记忆** - 代码变更后需要同步更新 CLAUDE.md
2. **记录架构决策** - 做出重要技术决策需要创建 ADR (Architecture Decision Record)
3. **维护文档索引** - 新增或修改技术文档需要更新索引
4. **会话结束检查** - 用户结束会话前检查是否需要持久化信息
5. **用户手动触发** - 用户执行 `/update-memory` 或 `/record-refactor`

## 工作流程

### 流程图

```
┌─────────────────────────────────────────────────────────────┐
│                    文档更新工作流程                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 检测变更                                                 │
│     ├── 是否有新的架构决策？                                 │
│     ├── 是否有重要的代码变更？                               │
│     └── CLAUDE.md 状态是否需要更新？                         │
│                                                             │
│  2. 确定更新类型                                             │
│     ├── 增量更新 (推荐) - 仅修改受影响的部分                 │
│     └── 完整重建 - 仅当结构严重过时时                        │
│                                                             │
│  3. 执行更新                                                 │
│     ├── 读取现有 CLAUDE.md                                   │
│     ├── 应用增量修改                                         │
│     └── 更新时间戳                                           │
│                                                             │
│  4. 维护索引 (如需要)                                        │
│     ├── 更新 Docs/index/_tree.json                          │
│     └── 更新 Docs/index/index.md                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 增量更新策略

**优先使用增量更新**，遵循以下原则：

1. **最小改动** - 只修改必要的部分
2. **保持结构** - 不改变文档的整体格式
3. **追加优先** - 新内容追加到对应章节末尾
4. **时间戳更新** - 每次更新必须修改最后更新时间

### 更新触发条件

#### [文档创建](#文档创建)

当根据上下文如果有以下内容时，那么进行文档创建

- 新增函数(新模块)
- 新增API
- 新增功能

#### [文档更新](#更新文档)

当根据上下文发现有以下内容时，进行文档更新

- 函数更替
- 模块内新增函数(已有模块)
- 功能更改
- 函数参数变化
- 结构变更

#### [CLAUDE.md更新](#claudemd更新)

当根据上下文发现有以下内容时，进行CLAUDE.md更新(创建)

- 不存在CLAUDE.md时
- 用户明显偏好(比如用户明显生气，用户明显情绪低落时)
- 文档较大更新，文档创建
- 生成项目级todolist时
- 架构决策时
- 用户明确要求时
- 用户明确指出项目约束时候


#### [索引维护](#索引维护)

当根据上下文发现有以下内容时，进行索引维护

- 新增技术文档（API/模块文档）
- 删除技术文档
- 文档分类变更
- 文档描述需要更新
- 项目结构发生变化

## CLAUDE.md更新

### 判断是否需要创建 CLAUDE.md

如果项目根目录不存在 CLAUDE.md 文件，则需要进行创建

### 读取 CLAUDE.md 模板

参考 [CLAUDE.md 模板](templates\claude-md.template.md) 进行创建或更新

### 确定更新内容

根据触发条件，确定需要更新的章节：

| 触发条件 | 更新章节 |
|---------|---------|
| 架构决策 | 架构决策记录 (ADR) |
| 新增模块/API | 技术文档索引 |
| 任务状态变化 | 当前任务状态 |
| 用户偏好变化 | 用户偏好 |
| 项目约束变化 | 约束条件/禁止项 |

### 执行增量更新

1. 读取现有 CLAUDE.md 内容
2. 定位需要更新的章节
3. 应用最小改动原则：
   - 追加新内容到章节末尾
   - 仅修改受影响的行
   - 保持现有格式不变
4. 更新文件末尾的时间戳

### 验证更新结果

确保更新后的 CLAUDE.md：
- 格式正确，Markdown 语法有效
- 链接可访问
- 表格对齐

## 文档创建

### 根据需求确定文档创建类型

根据上下文确认需要确认文档的类型，如确定更新的是API还是普通的功能函数
```
API -> 创建API文档

新增功能函数 ->根据分类(一般由函数所在类决定)来创建模块文档
```

### 创建文档

#### 文档命名以及储存规则

一般的，参考代码结构进行命名

eg:
```
对应的类/abc/abcd/abcdef.jar

对应的文档/Docs/{文档类型}/abc/abcd/abcdef.md
```

参考[文档模板](#文档模板)根据上一步确定的类型，必须阅读对应的文档类型的模板来进行创建

### 更新CLAUDE.md

将新增的文档作为索引，更新至CLAUDE.md中的"技术文档索引"段落其中必须包含

- 文档摘要
- 文档地址

## 文档更新

### 查找是否存在文档

使用`doc-navigation`SKILL根据上下文定位到已有旧文档，如果没有文档的话，那么在确保情况属实的情况下，进行[文档创建](#创建文档)

### 更新文档

参考[文档模板](#文档模板)对已有文档进行局部更新

### 判断是否需要更改摘要以及储存位置

#### 更改CLAUDE.md 中的文档摘要

只有发生了以下情况，你才需要更改摘要

- 整个类/Api进行了重构
- 原有摘要和当前文档不匹配

#### 更改储存位置

只有当发生了以下情况，你才需要更改储存位置

- 代码储存位置进行了变更

如果满足了上述要求，你必须要完成以下两项任务

- 更改文档储存地址

  参考
  ```
  对应的类/abc/abcd/abcdef.jar

  对应的文档/Docs/{文档类型}/abc/abcd/abcdef.md
  ```
  来更新位置
  
- 更新CLAUDE.md 索引文档位置

## 索引维护

### 索引文件结构

项目使用双格式索引：JSON + Markdown

```
Docs/
├── index/
│   ├── _tree.json    # 结构化索引 (机器可读)
│   └── index.md      # 可读索引 (人类可读)
└── {category}/
    └── *.md
```

### _tree.json 结构规范

#### 分类依据

- 项目结构 - 项目的文件夹一般就代表着一些类别，所以我们可以参考项目结构作为分类依据

```json
{
  "version": "1.0",
  "updated": "YYYY-MM-DD",
  "categories": [
    {
      "name": "分类名称",
      "path": "Docs/category/",
      "description": "分类描述",
      "documents": [
        {
          "name": "文档名称",
          "file": "filename.md",
          "description": "文档描述",
          "tags": ["tag1", "tag2"]
        }
      ]
    }
  ]
}
```

### 索引更新规则

1. **新增文档** - 在对应分类的 `documents` 数组中添加条目
2. **删除文档** - 从数组中移除条目
3. **修改文档** - 仅更新描述或标签，不修改文件名
4. **新增分类** - 在 `categories` 数组中添加新分类对象

## 最佳实践

### DO (推荐)

- ✅ 每次重要变更后更新 CLAUDE.md
- ✅ 使用增量更新而非重写整个文件
- ✅ 为架构决策创建 ADR
- ✅ 保持文档索引与实际文件同步
- ✅ 更新时同步修改时间戳

### DON'T (避免)

- ❌ 在 CLAUDE.md 中存储敏感信息
- ❌ 频繁重写整个文档
- ❌ 忽略文档索引的维护
- ❌ 创建重复的 ADR 条目
- ❌ 忘记更新时间戳
